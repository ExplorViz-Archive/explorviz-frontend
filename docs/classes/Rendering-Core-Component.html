    Rendering-Core-Component Class

    Extends Ember.Component
    Defined in: app\components\visualization\rendering\rendering-core.js:9
    Module: visualization.rendering
    Parent Module: explorviz

    This component contains the core mechanics of the different (three.js-based) renderer. All functions below are called in a determined order, hence you only need to override them in your custom renderer.

    See Landscape-Rendering or Application-Rendering for example usage.

    cleanAndUpdateScene

    ()

    Defined in app\components\visualization\rendering\rendering-core.js:435

    Inherit this function to update the scene with a new renderingModel. It automatically removes every mesh from the scene and finally calls the (overridden) "populateScene" function. Add your custom code as shown in landscape-rendering.

    Defined in app\components\visualization\rendering\rendering-core.js:404

    Defined in app\components\visualization\rendering\rendering-core.js:65

    This overridden Ember Component lifecycle hook enables calling ExplorViz's setup code for actual rendering and custom listeners.

    initRenderings

    Defined in app\components\visualization\rendering\rendering-core.js:91

    This function is called once on the didRender event. Inherit this function to call other important function, e.g. "initInteraction" as shown in Landscape-Rendering/initInteraction:method.

    populateScene

    ()

    Defined in app\components\visualization\rendering\rendering-core.js:388

    This function is called once on initRendering and everytime at the end of "cleanAndUpdateScene". Inherit this function to insert objects in the Three.js scene. Have a look at Landscape-Rendering/cleanAndUpdateScene:methodor Application-Rendering/cleanAndUpdateScene:methodfor examplary usage.

    preProcessEntity

    ()

    Defined in app\components\visualization\rendering\rendering-core.js:467

    This function is called automatically when a new landscape was fetched. It is executed before Rendering-Core/cleanAndUpdateScene:method. Inherit this function to preprocess the Landscape for rendering, e.g. filter some value.

    See Application-Rendering for example usage.
