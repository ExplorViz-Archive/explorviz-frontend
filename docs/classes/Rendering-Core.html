    Rendering-Core Class

    Extends Ember.Component
    Defined in: app/components/rendering-core.js:3

    This component contains the core mechanics of the different (three.js-based) renderer. All functions below are called in a determined order, hence you only need to override them in your custom renderer.

    See Landscape-Rendering or Application-Rendering for example usage.

    Defined in app/components/rendering-core.js:145

    Inherit this function to update the scene with a new renderingModel. It automatically removes every mesh from the scene. Add your custom code as shown in landscape-rendering.

    cleanup

    Defined in app/components/rendering-core.js:128

    This function is called when the destroy event is fired. Inherit this function to cleanup custom properties or unbind listener as shown in Landscape-Rendering.

    initRendering

    ()

    Defined in app/components/rendering-core.js:61

    This function is called once on the didRender event. Inherit this function to call other important function, e.g. initInteraction as shown in {@landscape-rendering}.

    populateScene

    ()

    Defined in app/components/rendering-core.js:118

    This function is called once on initRendering. Inherit this function to insert objects in the Three.js scene. Have a look at Landscape-Rendering for an example.

    preProcessEntity

    ()

    Defined in app/components/rendering-core.js:166

    This function is called automatically when a new landscape was fetched. It is executed before cleanAndUpdateScene. Inherit this function to preprocess the Landscape for rendering, e.g. filter some value.

    See Application-Rendering for example usage.
